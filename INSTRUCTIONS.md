## プロジェクト概要

ゲームパッド入力を 14 ビット MIDI CC（コントロールチェンジ）メッセージに変換する Python アプリケーションです。接続されたゲームパッドのアナログスティック位置を読み取り、高分解能の MIDI データを仮想または物理 MIDI デバイスへ送信します。

## アーキテクチャ

### 主要コンポーネント

- **`midi_simulator.py`** - `GamepadMidiController` クラスを持つメインアプリケーション。以下を担当：
  - 起動時のインタラクティブな MIDI ポート選択
  - インタラクティブなゲームパッド選択（複数接続時）
  - pygame によるゲームパッドの初期化と入力処理
  - python-rtmidi による MIDI 出力
  - 14 ビット CC メッセージの生成と送信
  - デッドゾーン処理付きのリアルタイム入力処理

- **`setup.py`** - 依存関係を確認し、パッケージをインストールし、シミュレーターを起動できる自動セットアップスクリプト

### MIDI マッピング

ゲームパッドの各軸に対する MIDI CC マッピング：
- 左スティック: X→CC#16/48 (MSB/LSB)、Y→CC#17/49 (MSB/LSB)
- 右スティック: X→CC#18/50 (MSB/LSB)、Y→CC#19/51 (MSB/LSB)
- 14 ビット分解能: 0-16383（中立位置は 8192）
- 送信順序: LSB を先、MSB を後
- ボタン: ゲームパッドのボタン i → CC#(20+i)、i=0..9（127=押下 / 0=離上、受信側のしきい値は 64）
- 状態セレクタ: ショルダーボタン (LB/RB) で 0..16 を増減 → CC#30（0-127 にスケール）

## 開発環境

**重要: このプロジェクトは Python 仮想環境 (venv) を使用します。Python コマンドを実行する前に必ず仮想環境を有効化してください。**

### 仮想環境のセットアップ（必須）
```bash
# 仮想環境を作成
python -m venv .venv

# 有効化 (Windows)
.venv\Scripts\activate

# 有効化 (macOS/Linux)
source .venv/bin/activate

# 無効化
deactivate
```

### 仮想環境の利用ルール
- Python コマンド実行前に**必ず** `.venv` を有効化する
- パッケージをグローバルに**インストールしない** - 仮想環境を使う
- コマンドプロンプトに `(.venv)` が表示されているかで有効状態を確認する
- 開発・テストはすべて仮想環境内で行う

### パッケージ管理
```bash
# 先に仮想環境を有効化 (Windows)
.venv\Scripts\activate

# 依存関係をインストール
pip install -r requirements.txt

# 不足している依存関係を確認
python setup.py
```

### アプリケーションの実行
```bash
# 先に仮想環境を有効化 (Windows)
.venv\Scripts\activate

# 直接実行
python midi_simulator.py

# セットアップスクリプト経由（インタラクティブ）
python setup.py
```

## 依存関係

- **pygame>=2.0.0** - ゲームパッド入力処理
- **python-rtmidi>=1.4.0** - MIDI 出力
- **Python 3.7+** - 実行要件

## 主要な設定

- **DEADZONE**: アナログ入力のしきい値（デフォルト: 0.1）
- **更新レート**: 100Hz（メインループで 10ms スリープ）
- **MIDI CC 番号**: `GamepadMidiController` のクラス定数として定義
- **14 ビット範囲**: 0-16383（中立位置は 8192）

## UI 機能

起動時にインタラクティブなデバイス選択を提供します：
- **モード選択**: 起動時に通常モード（ゲームパッド）かデモモードを選択。デモモードはゲームパッド不要で、スティック（円運動）・ボタン（順次）・状態（0↔16 スイープ）を継続送信し、受信側テストに使える。
- **MIDI ポート選択**: 利用可能な MIDI ポートを一覧表示し、ユーザーが選択
- **ゲームパッド選択**: 単一なら自動選択、複数ならメニュー表示
- **エラー処理**: デバイス接続失敗を穏当にハンドリング

## MIDI デバイスのセットアップ

アプリケーションは既存の MIDI ポートに接続します。loopMIDI と DAW の詳細なセットアップ手順は README.md を参照してください。

## コードパターン

- 入力処理は変化検出を用いて MIDI トラフィックを最小化する
- 14 ビット値は 7 ビットの MSB/LSB ペアに分割する
- pygame と MIDI リソースを穏当にクリーンアップする
- デバイス初期化失敗に対する例外処理
